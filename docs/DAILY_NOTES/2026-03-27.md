## Evening Session — Bug Fixes Deployed

Stephen reported 3 issues and asked for careful, slow fixes:

### Fix 1: Input fields not appearing on Canvas
**Problem:** Business Model section (9 canvas textareas) was missing from the Input tab, so users had no way to enter data that would appear on the Canvas.
**Fix:** Restored the Business Model section with all 9 textareas (Problem, Solution, UVP, Unfair Advantage, Customer Segments, Key Metrics, Channels, Revenue Streams, Cost Structure) to the Input tab. Values flow to Canvas via shared `canvas` state and `saveCanvasCell()`.

### Fix 2: Canvas should have NO AI integrations
**Problem:** Canvas tab had AI-generated content, loading spinners, and score badges.
**Fix:** Removed `await genCanvas()` from `submit()`. Canvas tab now shows 9 editable textareas with no AI, no loading screen, no score badges. Submit just saves profile and switches to Canvas tab.

### Fix 3: Corrupted characters on loading screen
**Problem:** Non-ASCII characters (NEL `\u0085`, curly quotes, em dashes) were corrupting the build.
**Fix:** Stripped all non-ASCII characters from App.jsx. File is now pure ASCII (69228 bytes). All quotes are straight, all dashes are `--`.
